Additional assertion functions¶

testmania.deep.assert_deep_equal(obj1, obj2, message=None, partial_dict_match=False)¶

Test for equality two deeply nested data structures of simple types like dicts or lists.

Provides readable message if they don’t match with path containing found difference.

>>> obj1 = {
...    'foo': 1,
...    'bar': [
...         {'baz': 'qux'},
...         {'bam': 'qix', 'tee': 'uup'},
...    ],
...    'wee': 3,
... }

>>> obj2 = {
...    'foo': 1,
...    'bar': [
...         {'baz': 'qux'},
...         {'bam': 'qix', 'tee': 'uop'},
...    ],
...    'wee': 3,
... }

>>> assert_deep_equal(obj1, obj2)
Traceback (most recent call last):
    ...
AssertionError: 
---------- Left ----------
{'bar': [{'baz': 'qux'}, {'bam': 'qix', 'tee': 'uup'}], 'foo': 1, 'wee': 3}
---------- Right ---------
{'bar': [{'baz': 'qux'}, {'bam': 'qix', 'tee': 'uop'}], 'foo': 1, 'wee': 3}

at /bar.1.tee, left has value 'uup', right has value 'uop'

Objects to compare may be arbitrary nested structures of lists, dicts, UserDict, UserList or inherited of thereof such as defaultdict or custom user data types.

If partial_dict_match=True, dicts in obj1 are allowed to be a superset of corresponding dicts in obj2 at any level, e.g. to have extra keys. This is handy in cases when it is important to test whether at least necessary items are present in a data structure.

testmania.xml.assert_xml_equal(actual, expected, msg=None, ignore_whitespace=True, ignore_extra_elements=False, ignore_element_order=False, ignore_list_order=False, ignore_extra_attrs=False)¶

Test that two XMLs are equivalent.

Provides detailed message if they don’t match.

actual and expected should be strings with xmls to test.

Parameters:
  • ignore_whitespace – whether insignificant whitespace should be ignored
  • ignore_extra_elements – whether actual is allowed to have extra elements with tag names not present in expected on same level. Set to True if you’re just testing presence of necessary elements.
  • ignore_element_order – controls whether order of elements with distinct tag names is important on a level.
  • ignore_list_order – controls whether order of elements with same tag names is important on a level. Set to True if recurring elements represent sets, not lists in your XML.
  • ignore_extra_attrs – whether actual is allowed to have element attributes that aren’t present in expected.
